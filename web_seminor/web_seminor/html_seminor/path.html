Abstruct

DOSの黒い画面にカタカタしていた時代は、頻繁に触る人なんかでは、結構知っている方が多いのですが、今の時代はあまり打ち込む機会がないですから、知っている範囲が限られてしまいますね

そういう人達のために、今回は知っておいて困らないパスの詳しい使い方です

相対パス編

相対パスとは自分自身のファイルのある位置を中心に位置を指定する方法です、主に自分のサイト内へリンクを張る場合に使用されます

同じフォルダにあるファイルの場合は


<a href="ファイル名">リンク</a>

と書きますが、自分より下のディレクトリのファイルを取ってくる場合は


<a href="ディレクトリ名/ファイル名">リンク</a>

深いディレクトリのファイルを取ってくる場合は


<a href="ディレクトリ１/ディレクトリ２/ファイル名">リンク</a>

とディレクトリ名をスラッシュ("/")で区切ってどんどん書いていきます

ここまでは一般的に知られた方法ですが、ここからがちょっと味付け

ここまでの方法では、自分より下の階層のファイルしか取ってこれませんが、もし、自分より上の階層のファイルを取ってきたい場合ってどうすればいいのでしょう？

こういう場合は


<a href="../ファイル名">リンク</a>

と言う風に書きます。../と書いた場合、上のフォルダを指定したことになるんですね(当然２回書けば２つ上のフォルダを指定出来る)

ちなみに、これに対して./と書いた場合は、現在のフォルダを指定していることになります、普通は省略しますけど、同じフォルダにあることをハッキリ言っておきたい場合に一応用途があります

絶対パス編

http://...と始まる物が、一般的には絶対パスと言われる物ですね

でも、コレでは正解の半分、他にもう一つ絶対パスの指定方法があって


<a href="/index.html">topへ</a>

この様に一番最初をスラッシュ("/")で始めると、サイト内の如何なる所からでもindexhtmlへジャンプ出来ます。便宜的にサイト内絶対パスと読んでおきましょうか

コンテンツを整理しているとき、ファイルの移動の可能性があることを考えると相対パスでindex.htmlへリンクを張ると都合が悪いですよね

そういう相対パスでは都合の悪い場所で使い道があります
